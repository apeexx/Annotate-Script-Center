# 阿里 LabelX 快判 AI 官方判断规范整理（0422口径）

本文件是人工可读版规范；模型可读规则继续以 `rules-v2.ai.md` 等文件为准。

## 任务目标

- 从语音助手用户视角，在 `asr_text1` 与 `asr_text2` 中判断更优候选。
- AI 仅提供参考建议，不自动保存、提交、领取、流转。

## P0 / P1 / P2 分层

- P0（硬伤）：核心实意词/专有名词/动作词/否定词错误，导致语义或用户意图改变。
- P1（硬伤）：漏转、多转、少字、多字导致语义完整性明显下降。
- P2（轻微）：标点、语气词、儿化音、轻微格式差异，不影响语义和体验。

决策优先级：

1. 一条有 P0/P1，另一条仅 P2 或无错 -> 选无 P0/P1 的一条。
2. 两条均有 P0/P1 且都影响理解 -> `both_bad`。
3. 两条都无 P0/P1 -> 选更实用、自然、贴合用户意图的一条；少用 `uncertain_or_similar`。

## 关键优先级

- 实意词、专有名词、动作词、否定词优先级高于标点、空格、格式。
- 听音结果是辅助证据，不是直接替代候选文本的最终答案。
- 上文与 Web Search 仅作消歧，不能覆盖音频事实与候选比较。

## both_bad / uncertain_or_similar 使用限制

- `both_bad` 只在两条都明显不合格且影响理解时使用，不作为“模型没把握”兜底。
- `uncertain_or_similar` 仅在两条都合格且无明显优劣时使用（如“他/她/它”缺上下文）。

## Web Search 辅助规则

- 人名、地名、品牌、作品名、产品名、术语、行业词优先用 Web Search 辅助确认。
- 搜索能确认一个候选为真实固定词时，优先真实固定词。
- 若两个候选都不成立且可确认正确词另有其词，可判 `both_bad`。

## 近期错例摘要（标准答案）

1. 共同核心漏字（两条都漏“十字路口”关键字）-> `both_bad`
2. 重复次数差异（“确认确认确认” vs “确认确认确认确认”）-> 更接近音频者
3. 动作实词双错（爬/趴/拿等关键动作词都错）-> `both_bad`
4. 实意词优先于格式（“主题解说” vs “主题转述”）-> 实意词正确者
5. 领域词错切语气词（“暴击装呗” vs “暴击装备”）-> 领域词正确者
6. 否定/肯定语义相反（“不行” vs “就是”）-> 语义正确者
