# 题卡选中与上下题导航

## 请求标识 / 目的

该记录用于说明详情页中题卡选中、鼠标切换题卡以及快捷键上下题行为是否会触发业务接口。

## 页面入口 / 触发动作

- 在详情页当前页内，鼠标点击第 5 条题卡。
- 在页面 `body` 聚焦状态下按 `S`、`W`。
- 在页面 `body` 聚焦状态下按 `Shift+ArrowDown`、`Shift+ArrowUp`。

## 请求摘要

本次操作未观察到新增业务 XHR/fetch 请求。

仅页面停留期间可能继续出现周期性心跳：

```text
POST /api/v1/label/center/timer
```

该心跳与题卡选中或快捷键操作没有直接业务关联。

## 请求体摘要

- 当前记录未见独立 request body；以路径参数或 query 为主。

## 响应摘要

- 当前文件未补充独立响应结构。

## 关键字段

- 题卡当前选中态主要是前端 DOM 状态。
- 单纯选中题卡不是保存行为，不会调用 `POST /subTask/{subTaskId}/data`。
- 上下题快捷键是否生效可能依赖页面内部焦点、滚动容器或实际用户键盘事件上下文；本次只确认自动化按键未产生业务请求。
- 用户后续确认真实用户快捷键上下题复测也不会产生业务请求，因此不再单独采集该分支的 Network。

## 前端接入建议

- 扩展如需识别当前题卡，可读取 `.labelRender-item-selected`。
- 不要把题卡选中变化当作答案保存或页面数据切换信号。
- 对快捷键支持应做容错：以 DOM selected 变化为准，而不是仅监听键盘事件。
- 如果后续需要可靠“当前题号”，建议结合页面显示的 `第 N 题` 与 selected 题卡位置。

## 风险 / 未确认项

- 页面底部或其他布局下是否存在独立的上一题/下一题按钮，本次未观察到业务请求；后续无明确需求时不再复测。
