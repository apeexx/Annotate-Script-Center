# Aishell Tech 越南语助手运行时

## 当前范围

- 仅在 `https://mark.aishelltech.com/mytask/mark?...` 生效，当前同时兼容默认标注场景与 `scene=check` 质检场景。
- 平台前端显示名统一使用 `希尔贝壳`；不改内部平台 ID、目录名或接口路径。
- 当前为 Aishell Tech 平台下第二个正式脚本，与 `闽南语助手`、`泰语助手` 同平台互斥；生效脚本由 `platforms.aishellTech.activeScriptId` 控制。

## 当前能力

- 页面注入：
  - 原生“保存”按钮右侧注入 `AI识别`
  - 原生工具栏注入 `全部AI批量识别`、`未完成的AI批量识别`、`停止批量`
- 条目数据按页面场景读取：
  - 标注页：`packageItemList`
  - 质检页：`checkPackageItemList`
  - 质检记录使用 `markTaskItemId` 对齐原标注条目；稀疏业务序号不作为数组索引
- 结果区展示：
  - `原始文本`
  - `识别文本`
  - `当前语速`
  - `语速建议`
- 单条动作保留：
  - 复制识别文本
  - 填入并保存当前条
  - 忽略结果
- 只要 AI 返回了文本或语速建议，“填入并保存”按钮就保持可用。
- 当识别文本与页面文本、语速建议与页面语速都一致时，仍允许重新填入并保存；结果卡只显示非阻塞提示 `当前与页面一致，仍可重新填入保存。`。
- 批量链路继续复用 Aishell 现有模式：
  - AI 请求前端并发预取
  - 页面切条、填入 `text + speed`、点击真实保存按钮严格串行
  - 失败项保留详情与原始 JSON 展开
  - 批量状态区会继续显示并发 / 队列进度，并实时累计 `批量输入Token / 批量输出Token / 批量总Token / 批量预估人民币`
  - 质检页的“未完成”按 `checkStatus` 是否已有结果筛选；标注页继续使用原 `dataStatus` 口径

## AI 口径

- 只走单阶段 Omni 转写。
- 默认模型：`qwen3.5-omni-flash`。
- 成功结果固定包含 `recommendedText / recommendedSpeed / referenceText`，其中语速正式值为 `slow / normal / fast`。
- `当前识别结果` 当前会展示 `Token` 与 `预估人民币`；费用口径复用共享百炼价格配置。
- options 页只暴露：
  - 单模型选择
  - 单 Prompt
  - 共享高级参数
  - `AI 连续填入并发数量`
- `GET /defaults` 不可用时，options 当前会回退到本地完整单阶段默认值，继续展示默认模型、Prompt 与共享高级参数。
- 不接词表、不做转换/比较双阶段、不显示听音文本差异高亮。

## 越南语文本规范

- 保留正常词间空格与越南语重音字符。
- 清理首尾空格和重复空格。
- 去掉标点前多余空格。
- 统一标点后单空格。
- 句末缺少终止标点时补 `.`。
- 语速固定归一为英文枚举：`slow / normal / fast`；中文慢、中速/正常、快只作为兼容输入。
- 不再执行旧闽南语脚本的“去全部空格 + 补中文句号”规则。

## 保存边界

- 只有用户明确点击单条填入或批量入口后，才会触发页面真实“保存”按钮。
- 保存请求的 `mark` 固定写入 `{"text":"...","speed":"..."}`。
- 质检页只允许回填 `.mark-area` 文本与语速并点击该区域上方保存；不得操作 `.check-area` 的质检意见、合格/不合格或下方保存。
- 不自动提交任务。
- 不跨分包处理。
- 不绕过页面 disabled / 只读限制。
- 扩展重载后若页面出现旧上下文，运行时会提示刷新业务页面。
