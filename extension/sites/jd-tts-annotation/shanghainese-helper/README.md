# 京东 TTS 上海话文本回填助手运行时

## 适用范围

- 仅在 `https://tts-biaozhu-pub.jd.com/#/annotation/dataset/annotate` 生效。
- 脚本 ID：`jdTtsShanghaineseAssistant`；默认关闭。
- 同一平台只启用本助手时才会挂载页面按钮。

## 当前能力

- 页面已自行发出 `POST /annotation/get_utterance/` 后，主世界脚本仅被动读取成功响应中的当前语句身份和临时音频资源。
- 主世界只在隔离世界携带一次性 nonce 请求时下载当前完整 WAV；跨世界只传递二进制和 `utteranceId + checksum`，不传递资源地址。
- 用户点击“上海话识别”后，浏览器仅在后端健康检查通过后才把 WAV Data URL 发送到统一后端的异步 jobs 接口。
- AI 返回必须回显本次请求的 `utteranceId + checksum`；回填时以完整 WAV 内容一致性判断是否仍是同一整条音频。波形画段或由此产生的快照变化不会阻止回填；完整 WAV 已切换时不写入旧结果。成功后只将后端按确认别名加工的 `listenText` 回填到点击时锁定的“文本:”或“文本：”textarea，并只派发一次 `input` 事件。
- jobs 完成态的外层 `data` 是服务端成功响应体，客户端会继续读取其内层业务 `data` 后再校验 `utteranceId + checksum`；不会把外层任务字段或 `jobId` 当作识别结果。
- 识别前必须在脚本中心首页填写并保存全局“AI 调用使用人”；保存操作会从扩展存储回读校验，只有回读值与输入一致才显示保存成功。
- 每次点击“上海话识别”都会重新读取当前扩展实例的使用人状态；缺失时停在“使用人检查”，不获取 WAV、不创建 AI job。扩展重新加载导致页面内容脚本失效时，信息卡会明确提示刷新当前标注页；存储不可读或当前实例未保存时会提示重新打开扩展首页并确认仅保留一个 `0.4.5` 实例。
- “后端健康检查”先探测当前保存的后端模式；当前端为服务端且 health 为 `404`、本机 health 可用时，只向本机创建一个 job。两端 health 均失败、`404` 或无法连接时，流程停在该步骤，不创建 job、不把 WAV 发送到 AI 后端。若 jobs 创建返回 `404`，信息卡会明确说明当前后端未部署上海话路由：本地模式提示启动本机统一后端，服务端模式提示远端尚未部署该路由；不显示完整地址、密钥或音频数据。
- “文本:”或“文本：”所在 `.cell` 中仅存在一个 textarea 时，固定显示“上海话 AI 信息”卡；另兼容已采样的共享结构：同一 `FONT` 内各有一个且顺序为“文本:”/“文本：”后接“拼音:”/“拼音：”，并且 `.cell` 直属的两个 Element UI 输入容器中第一个容器唯一含有 textarea。共享结构只查询、锁定和写入第一个容器的 textarea；绝不查询、读取、监听、写入或触发第二个（拼音）textarea。其他 textarea 不唯一或结构不完整的情况均不挂载、不回填。卡片默认展示 AI 使用人、当前状态、当前步骤、百炼原文、正字加工后文本、替换次数和回填结果；展开后只展示模型、总耗时、Token、预估人民币、排队等待、缓存命中和 requestId。
- 单次流程的状态顺序为：使用人检查、获取当前 WAV、后端健康检查、创建识别任务、等待识别结果、校验完整 WAV、写入文本框。失败卡仅显示失败步骤、脱敏错误摘要和下一步建议；超时、空识别、完整 WAV 切换、后端结果身份不一致和文本框不可写均有独立结论。
- 信息卡和扩展按钮会随 Vue 重建重新挂载；按钮仍优先在原生“自动标注”右侧，工具栏暂时不存在时才回退到“文本:”或“文本：”旁。
- 脚本中心的上海话详情页显示独立 AI 设置：AI 开关、固定 `60000ms` 超时、固定关闭 thinking、`qwen3.5-omni-plus / qwen3.5-omni-flash` 模型、可选 Prompt，以及 temperature、top_p、token、penalty、seed、stop 参数。保存后继续复用既有后端 defaults 和运行时读取链路。

## 明确不做

- 不读取、监听、填写或校验“拼音:”textarea。
- 不自动点击、聚焦、失焦、保存、提交、领取或切换下一条。
- 不调用平台的 `reserve/`、文本写入、保存或提交接口。
- 不做批量识别、音频分段、波形画段解析、普通话转换、繁简转换、文本润色或二次模型校正；仅在后端按人工确认的别名进行确定性正字替换。
- 不提供批量保存、快捷键或自动提交能力。
- 不把临时签名音频地址、WAV Data URL、Cookie 或 Authorization 写入 storage、结果卡、日志或文档。
- 不在信息卡显示 jobId、临时资源地址、音频内容、Cookie、Authorization 或其他鉴权信息。

## 安全校验

- 每次获得新语句身份时立即废弃旧音频快照；回填前若身份变化，只重新读取最新完整 WAV 并比较音频内容。相同完整 WAV 允许回填，不同完整 WAV 拒绝回填。
- 空结果、请求取消、页面切换到不同完整 WAV、文本框被替换或 `disabled` / `readonly`、后端结果身份不一致或任一音频/AI 错误时均不写入文本框。
- 音频只接受 WAV MIME，大小上限 2 MiB；后端 JSON 请求体上限 3 MiB，AI job 从创建起最长 60 秒。

## 验证

- 本地验收时，先在仓库根目录运行 `node platform-resources/backend/server.js`，并确认 `http://127.0.0.1:3333/api/jd-tts-annotation/shanghainese-helper/ai/recommend/health` 返回 `200`；然后在脚本中心首页保存“本地”后端模式、`http://127.0.0.1:3333` 和“AI 调用使用人”，刷新目标标注页并启用“上海话助手”。
- 确认“上海话 AI 信息”卡默认可见；切换不同语句或 Vue 重建后，它仍跟随“文本:”或“文本：”区域，且“拼音:”或“拼音：”输入框保持不变。
- 成功识别后确认信息卡依次经过健康检查、创建任务、等待结果和“校验完整 WAV”，显示百炼原文、正字加工后文本与替换次数，记录“已回填文本”且仅“文本:”或“文本：”框更新；任意点选波形画段后重试仍应识别整条 WAV。停止本机后端后重试应停在“后端健康检查”，不创建 job。页面不得自动保存、提交或跳转。真实登录页面验收需要在用户浏览器中完成。
