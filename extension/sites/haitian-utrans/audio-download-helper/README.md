# uTrans 音频下载助手运行时

## 命中范围

- 站点：`http://*/*`
- 当前仅在以下详情页路由内实际生效：
  - `index.php?d=worker&c=work`

## 当前能力

- 在 uTrans 任务详情页右下角挂一个小型悬浮窗
- 悬浮窗只提供一个主按钮：`下载当前音频`
- 已接入扩展平台设置，当前脚本详情页只有一个开关：
  - 是否开启悬浮窗下载功能
- 点击后从当前页面上下文动态提取：
  - `projectid`
  - `batchid`
  - `audioid`
- 优先使用页面可见文件名作为下载名；拿不到时回退为 `audio-<audioid>.wav`
- 通过页内登录态请求当前音频，并重新包装为 `audio/wav` Blob 触发下载
- 下载桥接改为扩展独立脚本文件注入，避免被页面 CSP 拦截内联脚本

## 为什么不能直接打开原始音频链接

- 当前平台的音频接口虽然返回的是 WAV 二进制，但服务端响应头不正确
- 浏览器直接打开原始链接时，容易把返回体当作文本显示乱码
- 因此当前实现不采用“新标签直接打开链接”的主方案，而是走页内请求 + Blob 下载

## 下载逻辑边界

- 只处理当前详情页当前音频
- 关闭平台设置中的该开关后，不再显示悬浮下载窗
- 不做音频预览
- 不做在线播放
- 不做批量下载
- 不做自动下载
- 不做自动提交或任何平台写操作
- 不保存 cookie、token、authorization 或完整音频 URL

## 文件

- `content.js`
  - 详情页命中判断
  - 当前音频参数提取
  - 悬浮窗渲染
  - 页内下载桥接与状态提示
- `content.test.js`
  - 详情页命中判断
  - 下载参数组装
  - 文件名回退

## 真实浏览器验证

1. 在真实 Chrome / Edge 重新加载 unpacked extension。
2. 打开已登录的 uTrans 任务详情页，确认右下角出现 `音频下载` 悬浮窗。
3. 点击 `下载当前音频`，确认浏览器开始下载，并且文件后缀为 `.wav`。
4. 若页面参数不完整或登录态失效，确认面板内出现明确失败提示，且不会自动重试。
